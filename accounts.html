Django implementation notes:
Auth: Custom User model (AbstractBaseUser) with role field (learner / tutor / organisation / admin / developer). Profiles: LearnerProfile, TutorProfile, OrganisationProfile — each OneToOne with User. Role selection screen fires after email verification, before first dashboard load. Social auth: django-allauth with Google provider (Phase 2). All auth views use HTTPS. Password reset tokens expire in 24hrs (Django default). AWS SES handles all transactional email: verification, reset, welcome.
